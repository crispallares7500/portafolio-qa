Proyectos

Pagina Web – QA Manual y Automation

Participé en el aseguramiento de la calidad de una aplicación web ya desarrollada, trabajando en un equipo que seguía la metodología ágil Scrum. El trabajo se organizaba en sprints semanales gestionados con Jira, donde tomé parte activa en las reuniones de Sprint Planning, las dailys y las print Review al finalizar cada sprint.

Me encargué del diseño de las pruebas de las historias de usuario aplicando la metodología BDD, a partir de los criterios de aceptación. Para ello, elaboré escenarios en lenguaje Gherkin que facilitaban la comprensión por parte de todo el equipo y los gestioné mediante AsserThat, manteniendo la trazabilidad entre requisitos, pruebas y resultados dentro de Jira. Las pruebas se ejecutaron tanto en entornos web como móviles, registrando los resultados por historia de usuario.

Durante cada sprint realicé pruebas funcionales y, al finalizarlo, pruebas exploratorias para detectar posibles defectos no contemplados inicialmente. Todos los errores encontrados se documentaron en Jira, vinculándolos a su historia de usuario correspondiente para asegurar una trazabilidad completa entre requisitos, pruebas y defectos.

Al cierre de cada sprint preparé la release en Jira y documenté en Confluence toda la información relacionada con la entrega: plan de pruebas, resultados de ejecucción incluyendo la documentación completa del plan de pruebas, resultados y estado de calidad del producto.

Tras la release, identifiqué los flujos principales de la aplicación y se automatizaron las pruebas de regresión utilizando Playwright y Pytest con Python. Estas pruebas se integraron en un proceso de integración continua mediante GitHub Actions, configurado para ejecutar automáticamente la regresión semanalmente en alineación con el cierre de cada sprint.

Portafolio QA – Desarrolladora Web y QA en Testing de APIs

Desarrollé un portafolio web utilizando HTML5, CSS3 y JavaScript, integrando un formulario conectado a una API generada con Supabase. Se diseñaron las tablas de la base de datos y se realizaron consultas en SQL para la inserción y consulta de datos de prueba del formulario.

Realicé testing de APIs utilizando Postman, creando pruebas para las peticiones GET (consulta), POST (creación), PATCH (edición) y DELETE (eliminación). Se implementaron tests que validan códigos de respuesta positivos, negativos y de autenticación, así como la correcta estructura y contenido de las respuestas.

Además, se implementó un flujo de integración continua con GitHub Actions, configurando Newman para ejecutar automáticamente las pruebas creadas en Postman en cada cambio del repositorio.
